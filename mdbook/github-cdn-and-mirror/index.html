自动部署在github上的 mdbook 或者 hexo等 国内访问加速的方法

国内访问加速

如果是备案的域名 用国内cdn 加速就好了，没备案。。。那就香港主机反代，套cf

dnspod 境内cname指向到cdn 境外直接到 xxx.github.io

这个不用具体说

国内md源文件在线管理加速方法

github 挂梯子当然可以，但是不是所的环境都有梯子的，所以 利用国内镜像

gitee

目前镜像是免费的，但是 gitee 真的很懒，只是他起步比较早 私有库的话真的无所谓他家的

在线编辑 md文件 压根没有代码高亮，真不知道gitee咋想的！

md文件里面的外联 会被gitee 套前缀地址 这个倒是优化的停积极的！

另外 gitee 自动推送到 github的速度很慢，一个小的修改 有时候很快2-3分钟，有时候可能要10分钟

gitee 之前的不通知 直接取消pages外联事件 就更别提了，当然 Gitee禁图床不应当是“差评”它的理由，但是你三天两头故障，连个恢复进度也没有，社区反馈你让去qq群，qq群管理员装死，连个公告也没有

还有 动则 政策限制违规文件。。。

下载一个小文件，你突然没通知 不提醒，突然有一天就给一个html登录页

功能该来该去，原有的功能 突然就无通知直接割掉

gitcode

虽然 很多人 烦csdn，但是gitcode 真的还不错，界面清爽，目前也支持镜像功能，但是我配置很久一直是授权错误。

无论用ssh 还是 http令牌，随之放弃

coding

可以用 devop自动推送，但是。。自从被腾讯收购后。。。。

另外 coding 从 github 直接clone的话 有概率失败，网络不稳定

执行时间，也从以前的 每个月 3000分钟，逐步 被腾讯优化到 1000分钟，现在又变成300分钟了。。。

不过300分钟 勉强也够用。。 但是coding的在线编辑也有诸多问题，与github天差地别，堪称垃圾

如果你觉得还好，可以参考这个文章配置 coding自动同步到github

codeup

阿里收购后，看的迷糊。。。 不会用他的自动同步。。。
